🔧 Задание 1. Ранжирование продуктов по цене
Описание решения
Для ранжирования продуктов использована оконная функция RANK(). Она позволяет присвоить каждому продукту ранг внутри группы (типа продукта) в зависимости от цены. Функция RANK() выбрана потому, что при одинаковых ценах продукты получают одинаковый ранг, а следующий ранг пропускается, что корректно отражает реальное положение.

SQL-код
sql
-- Задание 1. Ранжирование продуктов по цене от дорогих к дешевым внутри каждого типа
SELECT 
    product_id,
    product_type,
    model,
    year,
    base_msrp,
    RANK() OVER (PARTITION BY product_type ORDER BY base_msrp DESC) as price_rank
FROM products
ORDER BY product_type, price_rank;
Ключевые элементы
Элемент	Описание
PARTITION BY product_type	Группировка по типам продуктов
ORDER BY base_msrp DESC	Сортировка от дорогих к дешевым
RANK()	Присваивание рангов с пропуском при равенстве
ORDER BY product_type, price_rank	Финальная сортировка результата

🔧 Задание 2. Анализ динамики регистрации клиентов по штатам
Описание решения
Для сравнения дат регистрации клиентов в одном штате использована оконная функция смещения LAG(). Она позволяет получить значение из предыдущей строки в рамках той же группы (штата) без использования самосоединения таблицы.

SQL-код
sql
-- Задание 2. Для каждого клиента вывести дату добавления и дату добавления 
-- предыдущего зарегистрированного клиента в том же штате
SELECT 
    customer_id,
    first_name,
    last_name,
    state,
    date_added::DATE as registration_date,
    LAG(date_added::DATE, 1) OVER (PARTITION BY state ORDER BY date_added) as prev_customer_date
FROM customers
ORDER BY state, date_added;
Ключевые элементы
Элемент	Описание
PARTITION BY state	Отдельные вычисления для каждого штата
ORDER BY date_added	Сортировка клиентов по дате регистрации
LAG(date_added, 1)	Получение даты из предыдущей строки
date_added::DATE	Приведение timestamp к типу DATE

🔧 Задание 3. Нарастающий итог продаж для дилера 10
Описание решения
Для расчета нарастающего итога сначала с помощью CTE (Common Table Expression) агрегированы продажи по дням. Затем оконная функция SUM() в режиме накопления (с ORDER BY) вычисляет нарастающую сумму от первого дня до текущего.

SQL-код
sql
-- Задание 3. Рассчитать нарастающий итог продаж по дням для дилера с dealership_id = 10
WITH daily_sales AS (
    SELECT 
        sales_transaction_date::DATE as sale_date,
        SUM(sales_amount) as daily_sum
    FROM sales
    WHERE dealership_id = 10
    GROUP BY 1
)
SELECT 
    sale_date,
    daily_sum,
    SUM(daily_sum) OVER (ORDER BY sale_date) as running_total
FROM daily_sales
ORDER BY sale_date;
Ключевые элементы
Элемент	Описание
WITH daily_sales AS (...)	CTE для агрегации продаж по дням
WHERE dealership_id = 10	Фильтр только для дилера 10
SUM(daily_sum) OVER (ORDER BY sale_date)	Вычисление нарастающего итога
ORDER BY sale_date	Сортировка по хронологии

🧠 Выводы
В ходе выполнения лабораторной работы были изучены и применены на практике:

Функции ранжирования — RANK() позволяет ранжировать строки внутри групп, корректно обрабатывая одинаковые значения.

Функции смещения — LAG() дает возможность сравнивать текущую строку с предыдущей без сложных самосоединений.

Агрегатные оконные функции — SUM() в сочетании с OVER (ORDER BY ...) позволяет вычислять нарастающие итоги, сохраняя все строки исходного набора данных.

Ключевые концепции оконных функций:

PARTITION BY — создает окна (группы) для вычислений

ORDER BY внутри OVER() — задает порядок обработки строк

CTE — удобный способ предварительной агрегации данных

Преимущество оконных функций перед обычной агрегацией с GROUP BY — сохранение всех строк исходной таблицы с добавлением вычисляемых полей, что особенно важно для аналитических отчетов и дашбордов.
