# Задание 1

### Вывести список проданных моделей (model) и дату продажи для всех продаж в штате `'CA'` (через дилера или клиента).



```sql
SELECT 
    p.model,
    s.sales_transaction_date
FROM sales s
JOIN products p ON s.product_id = p.product_id
JOIN customers c ON s.customer_id = c.customer_id
LEFT JOIN dealerships d ON s.dealership_id = d.dealership_id
WHERE c.state = 'CA' OR d.state = 'CA';
```

## Пояснение

В этом запросе используется несколько **JOIN** для объединения данных из разных таблиц.

Соединяемые таблицы:

* `sales` — содержит информацию о продажах
* `products` — содержит модели товаров
* `customers` — содержит информацию о клиентах
* `dealerships` — содержит дилерские центры

Типы соединений:

* **JOIN (INNER JOIN)** между `sales`, `products` и `customers` используется потому, что каждая продажа должна иметь товар и клиента.
* **LEFT JOIN** используется для `dealerships`, так как продажа может быть совершена **онлайн**, и тогда `dealership_id` будет `NULL`.

Условие:

```
c.state = 'CA' OR d.state = 'CA'
```

означает, что продажа произошла **либо клиенту из штата CA**, либо **через дилера из штата CA**.

# Задание 2

### Найти клиентов, которые купили те же модели товаров, что и клиент с `ID = 1`.


```sql
SELECT DISTINCT 
    c.customer_id,
    c.first_name,
    c.last_name
FROM customers c
JOIN sales s ON c.customer_id = s.customer_id
JOIN products p ON s.product_id = p.product_id
WHERE p.model IN (
    SELECT p2.model
    FROM sales s2
    JOIN products p2 ON s2.product_id = p2.product_id
    WHERE s2.customer_id = 1
)
AND c.customer_id <> 1;
```

## Пояснение

В этом задании используется **подзапрос (Subquery)**.

Логика работы:

1. Подзапрос сначала определяет **модели товаров, купленные клиентом с ID = 1**:

```sql
SELECT p2.model
FROM sales s2
JOIN products p2 ON s2.product_id = p2.product_id
WHERE s2.customer_id = 1
```

2. Основной запрос находит **других клиентов**, которые покупали эти же модели.

3. Используется оператор:

```
IN (подзапрос)
```

чтобы сравнить модели.

4. Условие

```
c.customer_id <> 1
```

исключает самого клиента №1.

5. `DISTINCT` используется для удаления возможных повторов клиентов.

# Задание 3

### В таблице `products` создать категорию цены:

* `< 500` → `'Low'`
* `500–2000` → `'Mid'`
* `> 2000` → `'High'`



```sql
SELECT 
    product_id,
    model,
    base_msrp,
    CASE
        WHEN base_msrp < 500 THEN 'Low'
        WHEN base_msrp BETWEEN 500 AND 2000 THEN 'Mid'
        ELSE 'High'
    END AS price_category
FROM products;
```

## Пояснение

В этом задании используется функция **CASE WHEN**, которая позволяет создавать **условную логику внутри SQL-запроса**.

Принцип работы:

```
CASE
WHEN условие THEN значение
```

Категории формируются следующим образом:

* если `base_msrp < 500` → категория **Low**
* если `base_msrp BETWEEN 500 AND 2000` → категория **Mid**
* если `base_msrp > 2000` → категория **High**

Результат выводится в новый вычисляемый столбец:

```
price_category
```

При этом сама таблица **не изменяется**, создается только результат выборки.
