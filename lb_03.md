# Задание 1

### Найти минимальную, максимальную и среднюю цену (`base_msrp`) всех продуктов.

```sql
SELECT 
    MIN(base_msrp) AS min_price,
    MAX(base_msrp) AS max_price,
    ROUND(AVG(base_msrp), 2) AS avg_price
FROM products;
```

### Пояснение

В данном запросе используются **агрегатные функции**, которые позволяют получить сводную статистику по данным таблицы.

Используемые функции:

* `MIN(base_msrp)` — возвращает **минимальную цену** среди всех товаров.
* `MAX(base_msrp)` — возвращает **максимальную цену**.
* `AVG(base_msrp)` — вычисляет **среднюю цену** всех товаров.

Функция `ROUND(..., 2)` используется для **округления среднего значения до двух знаков после запятой**, что делает результат более удобным для анализа.

Поскольку агрегатные функции применяются **ко всей таблице**, предложение `GROUP BY` в данном запросе не требуется.

# Задание 2

### Посчитать количество клиентов (`customer_id`) для каждого пола (`gender`).


```sql
SELECT 
    gender,
    COUNT(customer_id) AS total_customers
FROM customers
GROUP BY gender
ORDER BY total_customers DESC;
```
**Скриншот выполнения SELECT**
<img width="382" height="94" alt="image" src="https://github.com/user-attachments/assets/b2047ea9-ed56-4bee-88f4-47258101cb1b" />


### Пояснение

В этом запросе используется **группировка данных с помощью `GROUP BY`**.

Логика работы запроса:

1. Сначала данные таблицы `customers` разбиваются на группы по значению столбца `gender`.
2. Для каждой группы выполняется агрегатная функция `COUNT(customer_id)`, которая подсчитывает количество клиентов.

Используемые элементы запроса:

* `GROUP BY gender` — группирует клиентов по полу.
* `COUNT(customer_id)` — подсчитывает количество клиентов в каждой группе.
* `ORDER BY total_customers DESC` — сортирует результат по количеству клиентов в порядке убывания.

Таким образом можно быстро увидеть **распределение клиентов по полу**.

# Задание 3

### Вывести штаты (`state`), где суммарные продажи (`sales_amount`) превышают **10 000**.

```sql
SELECT 
    c.state,
    SUM(s.sales_amount) AS total_sales
FROM sales s
JOIN customers c ON s.customer_id = c.customer_id
GROUP BY c.state
HAVING SUM(s.sales_amount) > 10000
ORDER BY total_sales DESC;
```

**Скриншот выполнения SELECT**
<img width="327" height="599" alt="image" src="https://github.com/user-attachments/assets/1f186be9-4d21-43fb-9c53-969876041061" />


### Пояснение

В данном задании используется **агрегирование с последующей фильтрацией групп**.

Запрос выполняет следующие действия:

1. Таблица `sales` соединяется с таблицей `customers` через `JOIN`, чтобы получить информацию о штате клиента.
2. С помощью `GROUP BY c.state` данные группируются по штатам.
3. Функция `SUM(s.sales_amount)` вычисляет **общую сумму продаж для каждого штата**.
4. Предложение `HAVING` фильтрует только те группы, где сумма продаж **больше 10 000**.

Важно различать:

* `WHERE` — фильтрует строки **до группировки**
* `HAVING` — фильтрует **результаты после группировки**

В этом случае используется именно `HAVING`, так как условие проверяет результат агрегатной функции `SUM()`.
